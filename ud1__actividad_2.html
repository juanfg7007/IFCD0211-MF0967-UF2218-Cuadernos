UF2218. UD1. Actividad 2. Caso práctico: Etapas del proyecto software: GestPerson


Enunciado (Descripción)

En la empresa TECNOBIT se quiere planificar el desarrollo de una nueva aplicación denominada “GestPerson”.

GestPerson es el proyecto software que se implantará en TECNOBIT dentro de su apartado de Sistemas de Información. Esta aplicación ayuda a la hora de seleccionar el personal para determinados proyectos, para lo que se apoyará en información de sus empleados en una base de datos donde se encontrará la información relativa a distintos aspectos que pueden ayudar a la hora de la selección del mejor personal disponible en la empresa.

El desarrollo de esta aplicación tendrá lugar dentro de la propia empresa, aprovechando las capacidades y recursos que ofrece la misma. Se trata de una aplicación con arquitectura de Servicios Web que se pretende que esté integrada en la Intranet de la empresa para que pueda ser consultada en todo momento por cualquier miembro de TECNOBIT, dependiendo del rol que tenga dentro de la organización.

El objetivo principal del desarrollo de esta aplicación es disponer de un repositorio de profesionales en el que poder identificar y localizar a aquellas personas con un mayor conocimiento en los distintos aspectos en los que TECNOBIT ofrece sus servicios, tanto a sus clientes, como para cubrir las propias necesidades internas.


Preguntas / Actividades a realizar 

Pregunta 1. Detalla las tareas que se deberían incluir en cada etapa del ciclo de vida del proyecto GestPerson. Para ello, puedes hacer uso del esquema de actividades y tareas propuesto a continuación. Tiene el orden de los hitos desordenados, por lo que deberás organizarlos correctamente.

  • Análisis de Requisitos.
  • Diseño.
  • Implementación.
  • Banco de pruebas.
  • Implantación.
  • Fin del Proyecto (Hito).

Análisis de requisitos

El análisis de requisitos es la base del proyecto, ya que define qué se va a construir y cuáles son las expectativas. Un análisis exhaustivo y detallado evita malentendidos y asegura que todas las partes interesadas estén alineadas.

Tareas:

  • Reunión con stakeholders: organizar reuniones con los stakeholders (interesados) del proyecto para comprender sus necesidades y expectativas.
  • Identificación de requisitos: documentar los requisitos funcionales y no funcionales del sistema. Esto incluye qué funcionalidades debe tener la aplicación y los niveles de rendimiento, seguridad, usabilidad, etc.
  • Análisis de viabilidad: evaluar la viabilidad técnica, operativa y económica del proyecto.
  • Priorización de requisitos: clasificar los requisitos según su importancia y urgencia.
  • Especificación de requisitos: crear un documento de especificación de requisitos detallado.
  • Validación de requisitos: revisar y validar los requisitos con los stakeholders para asegurar que se ha comprendido correctamente sus necesidades. 

Diseño

El diseño proporciona el plano detallado del sistema que guiará la implementación. Un buen diseño asegura que el sistema sea escalable, mantenible y que cumpla con los requisitos establecidos.

Tareas:

  • Diseño de la arquitectura: definir la arquitectura del sistema, asegurando que soporta la integración con la intranet de Tecnobit y cumple con la arquitectura de servicios web.
  • Diseño de base de datos: crear el modelo de datos, incluyendo el esquema de la base de datos que almacenará la información de los empleados.
  • Diseño de interfaces de usuario: crear maquetas y prototipos de las interfaces de usuario, asegurando una experiencia de usuario óptima.
  • Diseño de componentes: definir los componentes y módulos del sistema, especificando cómo interactuarán entre sí.
  • Documentación de diseño: crear un documento de diseño detallado que describa la arquitectura, el diseño de la base de datos, las interfaces y los componentes.

Implementación

La implementación convierte el diseño en un sistema funcional. La calidad del código y la correcta integración de los componentes son esenciales para evitar problemas durante las etapas posteriores.

Tareas:

  • Configuración del entorno de desarrollo: establecer y configurar las herramientas y entornos de desarrollo necesarios.
  • Desarrollo de la base de datos: implementar el esquema de la base de datos y desarrollar los procedimientos almacenados necesarios.
  • Codificación de componentes: escribir el código para los diferentes componentes y módulos definidos en la fase de diseño.
  • Desarrollo de interfaces de usuario: implementar las interfaces de usuario basadas en los prototipos diseñados.
  • Integración de servicios web: desarrollar e integrar los servicios web necesarios para la comunicación entre los componentes del sistema y la intranet de Tecnobit.
  • Revisiones de código: realizar revisiones de código para asegurar la calidad y coherencia del mismo.

Banco de pruebas

Las pruebas son críticas para asegurar que el sistema es robusto, fiable y cumple con los requisitos especificados. Detectar y corregir errores en esta fase evita problemas mayores en producción.

Tareas:

  • Desarrollo de casos de prueba: crear casos de prueba detallados que cubran todos los requisitos funcionales y no funcionales.
  • Pruebas unitarias: realizar pruebas unitarias en cada componente desarrollado para asegurar que funcionan correctamente de manera aislada.
  • Pruebas de integración: realizar pruebas para verificar que los componentes interactúan correctamente entre sí.
  • Pruebas de sistema: ejecutar pruebas que cubran todo el sistema para asegurar que cumple con los requisitos especificados.
  • Pruebas de usuario: involucrar a los usuarios finales en pruebas para validar que el sistema cumple con sus expectativas y necesidades.
  • Documentación de resultados: documentar los resultados de las pruebas y cualquier problema encontrado para su resolución.

Implantación

La implantación es la fase donde el sistema se pone en uso. Una implantación cuidadosa y bien planificada minimiza el riesgo de problemas y asegura una transición suave para los usuarios.

Tareas:

  • Preparación del entorno de producción: configurar y preparar el entorno donde la aplicación será desplegada.
  • Despliegue del sistema: implementar la aplicación en el entorno de producción.
  • Migración de datos: transferir los datos necesarios a la nueva base de datos si es aplicable.
  • Verificación post-despliegue: realizar pruebas en el entorno de producción para asegurar que la aplicación funciona correctamente.
  • Formación de usuarios: proporcionar formación a los usuarios finales sobre cómo usar la aplicación.
  • Documentación del sistema: crear documentación técnica y de usuario para facilitar el uso y mantenimiento del sistema.

Fin del proyecto (hito)

El cierre formal del proyecto asegura que se cumplieron los objetivos y proporciona una base para futuras mejoras y mantenimiento.

Tareas:

  • Revisión del proyecto: evaluar el proyecto en su totalidad para identificar lecciones aprendidas y áreas de mejora.
  • Aceptación formal: obtener la aceptación formal del sistema por parte de los stakeholders.
  • Cierre del proyecto: documentar el cierre del proyecto y archivar toda la documentación relevante.
  • Planificación de mantenimiento: establecer un plan de mantenimiento para asegurar el soporte continuo y la evolución del sistema.
