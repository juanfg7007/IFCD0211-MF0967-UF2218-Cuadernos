UF2218. UD3. Actividad 3. Caso práctico: XOM para tratamiento de XML.


Enunciado (Descripción)

XOM es una API desarrollada específicamente para Java, que da soporte al tratamiento de XML. Esta API realiza labores de parseo, búsquedas, modificación, generación y serialización.

Descargándote XOM y tomando como documento XML de partida el documento adjunto “UF2218_UD3_Actividad3_Caso práctico AA”, realiza la siguiente actividad.


Preguntas / Actividades a realizar 

Pregunta 1. Sigue los siguientes pasos:

  1. Incluye API XOM en tu proyecto Java.
  2. Explica las peculiaridades y el uso de la librería XOM y haz una pequeña comparación con DOM.
  3. Utiliza el SAX Builder para generar el árbol de nodos XML.
  4. Recorre las etiquetas hijas de la etiqueta raíz. Muéstralas por consola.
  5. Obtén la etiqueta y su texto de entre las etiquetas hijas de una etiqueta en concreto. Por ejemplo, de la etiqueta raíz ‘cancion’. Muéstrala por consola.
  6. Busca, con XPATH, la etiqueta ‘cancion’. Muéstrala por consola.

Podrás descargar XOM en el siguiente enlace: http://www.xom.nu/

Inclusión de API XOM en proyecto Java

Peculiaridades y uso de la librería XOM y comparación con DOM

XOM es una API para el manejo de documentos XML en Java, enfocada en la simplicidad, la corrección y la robustez. Algunas de sus peculiaridades incluyen:

  • Simplicidad y corrección: XOM está diseñada para ser sencilla y garantizar la corrección de los documentos XML, eliminando automáticamente los datos incorrectos o malformados.

  • Inmutabilidad: muchos de los objetos en XOM son inmutables, lo que mejora la seguridad y la estabilidad del código.

  • Fácil de usar: la API de XOM es coherente y fácil de aprender para los desarrolladores Java.

Comparación con DOM:

  • API de programación: XOM es más fácil y más coherente de usar comparado con DOM, ya que evita muchas de las complicaciones y peculiaridades de DOM.

  • Performance: XOM puede ser más eficiente en términos de memoria y velocidad, pero esto depende del caso de uso específico.

  • Corrección: XOM enfatiza la corrección del documento XML, eliminando automáticamente los datos incorrectos. DOM permite más flexibilidad pero requiere una gestión manual de la corrección.

  • Inmutabilidad: XOM utiliza inmutabilidad para muchos de sus objetos, lo que puede reducir errores de programación y mejorar la seguridad del código. DOM no tiene esta característica.

Utilización de SAXBuilder para generar el árbol de nodos XML

Explicación del uso de SAXBuilder

SAXBuilder en XOM se usa para construir un árbol de nodos XML a partir de un archivo XML. Utiliza el parser SAX (Simple API for XML) para analizar el XML de manera secuencial. SAXBuilder convierte este procesamiento secuencial en un modelo en memoria que XOM puede manipular.

  • Eficiencia: SAXBuilder es eficiente en el manejo de documentos XML grandes debido a su procesamiento secuencial.

  • Simplicidad: facilita la construcción de un modelo de documento XML que se puede manipular fácilmente con la API de XOM.

  • Corrección: garantiza que el documento XML resultante sea correcto y bien formado.
